# HOTCUT · ComfyUI — облачные генерации с оплатой в рублях

Ноды подключают в ComfyUI облачные модели изображений HOTCUT. Локально ничего не качаем — генерация идёт на сервере, оплата огнями HOTCUT (рубли, без зарубежных карт).

**Главная фишка — [GPT Image 2](https://hotcut.ru):** чёткий текст, кириллица и лица, которые Flux / Qwen / SDXL делают плохо. Плюс редактирование изображений и удаление фона — всё в одном графе.

---

## 🔌 Установка нод

**Через ComfyUI Manager** (проще всего)

- Открыть **Manager → Custom Nodes Manager**, найти **HOTCUT** → Install → перезапустить ComfyUI.

**Вручную**

```bash
cd ComfyUI/custom_nodes
git clone https://github.com/Smyshnikof/comfyui-hotcut.git
cd comfyui-hotcut
pip install -r requirements.txt
```

После перезапуска ноды появятся в категории **HOTCUT** (правый клик по холсту → Add Node → HOTCUT).

---

## 🔑 Получение API-ключа

1. Зарегистрироваться и пополнить баланс огней — [hotcut.ru](https://hotcut.ru)
2. Открыть **Студию → API** — [hotcut.ru/studio/api-keys](https://hotcut.ru/studio/api-keys)
3. **Создать ключ** → скопировать `hc_live_...` (показывается **один раз**, потом не подсмотреть)
4. Вставить ключ в ноду **HOTCUT API**

**Чтобы ключ не утёк в сохранённый `workflow.json`** — оставьте в поле `api_key` значение `$ENV` и задайте переменную окружения перед запуском ComfyUI:

```bash
# Windows
setx HOTCUT_API_KEY hc_live_xxx

# macOS / Linux
export HOTCUT_API_KEY=hc_live_xxx
```

## 🧩 Ноды

| Нода | Что делает |
|------|------------|
| **HOTCUT API** | Ключ + адрес сервиса. Отдаёт `config` остальным нодам. Кнопка **«Проверить баланс 🔥»** показывает остаток без запуска графа. |
| **HOTCUT · GPT Image 2** | Text-to-image: промпт + разрешение + соотношение сторон → `IMAGE` |
| **HOTCUT · GPT Image 2 Edit** | Image-to-image: подаёте `IMAGE`, дорисовываете / меняете → `IMAGE`. Сочетается с локальными нодами в одном графе. |
| **HOTCUT · Удалить фон** | Подаёте `IMAGE` → PNG с прозрачностью |

## ⚙️ Как собрать граф

**Генерация по тексту:**

```
[HOTCUT API] —config→ [HOTCUT · GPT Image 2] —IMAGE→ [Preview Image]
```

**Редактирование своей картинки:**

```
[Load Image] —IMAGE→ ┐
                     [HOTCUT · GPT Image 2 Edit] —IMAGE→ [Preview Image]
[HOTCUT API] —config→ ┘
```

**Удаление фона:**

```
[Load Image] —IMAGE→ ┐
                     [HOTCUT · Удалить фон] —IMAGE→ [Save Image]
[HOTCUT API] —config→ ┘
```

Параметры **GPT Image 2**:

- **resolution** — `1K` / `2K` / `4K`
- **aspect_ratio** — `auto`, `1:1`, `16:9`, `9:16`, `4:3`, `3:2`, `21:9` и др.

> ⚠️ **Ограничения по разрешению:** `auto` работает только в **1K**, а `1:1` — максимум **2K**. Если выбрать выше, разрешение автоматически снижается до допустимого — и списывается огней по фактическому, более низкому. Для полноценных **2K/4K** берите конкретное соотношение (`16:9`, `4:3` и т.д.).

## 🔥 Стоимость

Списывается огнями HOTCUT, цена зависит только от разрешения:

| Разрешение | Огней за кадр |
|------------|---------------|
| 1K | ≈ 39 |
| 2K | ≈ 65 |
| 4K | ≈ 103 |
| Удаление фона | ≈ 10 |

Само соотношение сторон на цену не влияет — но из-за ограничений выше (`auto` → 1K, `1:1` → до 2K) эффективное разрешение может снизиться, и тогда спишется меньше. Превью на ноде **«≈ N 🔥»** уже учитывает это и показывает реальную сумму.

**Списание — строго по факту успешной генерации.** Если запрос не прошёл (ошибка сервиса или блокировка модерацией) — огни не списываются.
