# ComfyUI · HOTCUT

Облачные модели изображений прямо в графе ComfyUI с оплатой в рублях (огнями HOTCUT).
Первый wedge — **GPT Image 2**: чёткий текст, кириллица и лица, которые локальные
модели (Flux/Qwen/SDXL) делают плохо или никак.

## Ноды

| Нода | Что делает |
|------|------------|
| **HOTCUT API** | Конфиг: API-ключ + базовый URL. Отдаёт `config` для остальных нод. |
| **HOTCUT · GPT Image 2** | text-to-image → `IMAGE`. Промпт, resolution (1K/2K/4K), aspect ratio. |
| **HOTCUT · GPT Image 2 Edit** | image-to-image: берёт `IMAGE` на вход (локальный рендер) → правит/дорисовывает в облаке → `IMAGE`. Композиция local + cloud в одном графе. |
| **HOTCUT · Удалить фон** | Берёт `IMAGE` → убирает фон в облаке → `IMAGE` (PNG с прозрачностью). ~10 огней. |

На ноде **HOTCUT API** есть кнопка **«Проверить баланс 🔥»** — показывает остаток огней прямо на ноде, без запуска графа.

## Установка

**Через ComfyUI Manager:** найдите «HOTCUT» (после публикации в реестре) и установите.

**Вручную:**
```bash
cd ComfyUI/custom_nodes
git clone https://github.com/itsmyshnikov/comfyui-hotcut.git
cd comfyui-hotcut
pip install -r requirements.txt
```
Перезапустите ComfyUI. Ноды появятся в категории **HOTCUT**.

## Ключ

1. Зарегистрируйтесь на [hotcut.ru](https://hotcut.ru), пополните баланс огней.
2. **Студия → API → Создать ключ.** Скопируйте `hc_live_...` (показывается один раз).
3. Вставьте ключ в ноду **HOTCUT API**.

Чтобы ключ не попал в сохранённый `workflow.json`, оставьте в поле `api_key`
значение `$ENV` и задайте переменную окружения перед запуском ComfyUI:
```bash
export HOTCUT_API_KEY=hc_live_xxx   # Windows: setx HOTCUT_API_KEY hc_live_xxx
```

## Стоимость (огни)

Зависит только от `resolution`: **1K ≈ 39**, **2K ≈ 65**, **4K ≈ 103** огней за кадр.
Ограничения провайдера (применяются автоматически): `1:1` — не выше 2K, `auto` — только 1K.
Перед генерацией нода печатает в консоль, сколько спишется.

## Как это работает

Нода обращается к публичному API HOTCUT (`/api/v1/images`), который списывает огни
и проксирует запрос к провайдеру. Списание — строго по факту успешной генерации;
при ошибке/блокировке модерацией огни не списываются.

## Внешний вид

Ноды HOTCUT оформлены в фирменном стиле: тёмное тело, лого в заголовке и
превью стоимости «≈ N 🔥» прямо на ноде (пересчитывается при смене resolution).
Реализовано лёгким фронт-расширением (`web/hotcut.js`) через стабильные хуки
ComfyUI — без рескина графа, чтобы не ломаться на обновлениях.
